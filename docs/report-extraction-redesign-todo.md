# 财报提取与图谱映射重构待办

## 1. 报告提取拆分：第一个 Guide 怎么写

目标：AI 先只生成 `facts.json`，只记录财报原文披露的类别、数据和证据，不做图谱映射。

需要明确：

- `facts.json` 的字段结构：公司、报告、财务指标、披露业务/产品类别、关键供应链事实、时间线事件、来源引用。
- 披露类别怎么写：保留财报原文名称，不强行改成图谱节点名。
- 缺失数据怎么写：没有披露就填 `null` 或空数组，不用 AI 推断。
- 供应链事实怎么写：例如瓶颈、产品放量、客户预付款、采购放量、产能扩张、库存变化、毛利率压力、技术切换。
- 证据要求：每条事实必须有 `sourceText` 和 `sourceSection`。
- 禁止项：不允许引用 `nodes.json` / `edges.json`，不允许写置信度，不允许写图谱节点或边的结论。

## 2. 图谱映射：第二个 Guide 怎么写

目标：AI 或脚本基于 `facts.json`、`nodes.json`、`edges.json` 生成 `mapping.json`，讨论重点放在映射和置信度输入。

需要重点讨论：

- 产品类别如何映射到父节点或子节点。
- 下游类别如何映射到父节点或子节点。
- 边如何映射：允许父节点到父节点、父节点到子节点、子节点到父节点、子节点到子节点。
- 图谱中没有对应节点时怎么进入 `unmappedFacts`，避免硬塞。
- 父节点数据是否下钻到子节点：默认不下钻，只记录父级汇总和是否有子节点提及/占比证据。
- 哪些由 AI 做：
  - 选择候选节点/边。
  - 抄写映射证据。
  - 标记是否明确提及、是否披露占比、是否披露产品级收入/增速。
- 哪些由脚本做：
  - 校验节点和边是否存在。
  - 计算毛利、毛利率等可由公式得到的数据。
  - 根据固定表格计算节点置信度和边置信度。
  - 生成最终 `extracted.json` 或前端可读的派生数据。
- 置信度需要拆开：
  - 节点置信度：财报增长代表该产品节点增长的程度。
  - 边置信度：该产品增长归因到某个下游的程度。
  - 最终边信号置信度 = 节点置信度 × 边置信度。

## 3. 前端如何做，数据如何汇总显示

需要设计：

- 财报详情页显示 `facts.json` 的原始披露信息：财务指标、披露类别、关键供应链事实。
- 图谱页面显示 `mapping.json` 或派生结果：节点信号、边信号、未映射事实。
- 合并态节点：显示父节点汇总数据。
- 展开态节点：父级虚线框显示汇总，子节点显示自己的披露/映射状态。
- 如果只有父节点数据：子节点显示“父级汇总未拆分”，不要自动复制父节点 YoY。
- 关键供应链事实需要可检索：未来新增节点或边时，可以反查哪些历史事实可能相关。
- 侧边栏和财报详情页需要区分：
  - 原文披露事实。
  - 图谱映射结果。
  - 脚本派生结果。

## 4. 小改动：线条文字

当前决定：

- 拓扑图边上的文字只保留简短 YoY 数字。
- 没有 YoY 的边不显示文字。
- 详细信息仍然放在悬浮提示和双击边的侧边栏里。
